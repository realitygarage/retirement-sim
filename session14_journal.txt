--- WHAT SHIPPED ---

v3.1.0 -- per-property dispositions, CPA liquidation defaults, settlement
gain-offset, 6th St STR group, HI-paydown knob.

Replaces the bespoke 6th-only sell path with a per-property disposition model
covering all three properties (6th, 15th, Lafayette). Introduces the Tartell
settlement scenario as first-class state (funded by sale proceeds; residual
routes to HI-debt paydown or invested cash). Adds STR income mode for the 6th
St primary (mirrors the duplex-top STR pattern, auto-stops on HI debt clear).

Source of truth: v3_phase1_consolidated_spec.md in project root.

   localStorage pins are silently dropped on load. Old JSON pin exports are
   unloadable. `sellYear`, `sixthSalePrice`, `sixthCostOfSale`, `sixthMTR`,
   `saleDrawFrac`, `keepPrimary` fields removed from DEFAULTS/SC_DEFAULTS.

   backs into depreciationTaken via `$ / DISPO_DEFAULTS.recaptureRate`.

3. Test scaffolding: `window.__engine` exposed for Playwright page.evaluate.

4. UI location: new "Dispositions & Settlement" panel in the Simulator tab,
   between "One-Time Decisions" and "Rental Operating Costs".

5. Barberry <-> Lafayette mapping: assumed correct per accountant sheet.
   Values/mortgages line up with BASE.lafayetteValue/Mortgage.

--- SEMANTIC NOTES / FLAG-IN-GLOSSARY ---

- `cashAst` floors at 0. Meaning: if the settlement is $525K but sale
  proceeds + prior savings don't cover it, the shortfall is assumed to be
  funded outside the sim (loan / retirement account / deferral). NW is not
  artificially depressed by phantom debt.

- STR income on the 6th St primary uses the same platform + clean + manager
  % as the duplex-top STR. Set to 0 to model self-managed operations.
  The tax impact on §121 (partial-home STR erodes future exclusion via
  non-excludable depreciation) is NOT modeled — flag for CPA.

- CA $1.2M cap on prior-1031 gain is allocated across barberry + fifteenth
  in year order (earliest sale claims cap first). If both are sold same
  year, the first key in [barberry, fifteenth] wins — should really be
  pro-rata; deferred to Phase 2.

- Depreciation-taken derived internally as `depreciationRecapture / 0.25`.
  If CPA sheet's recapture is ordinary-income-rated (not §1250), the back-
  into is wrong; single-slider UI can be updated if that becomes an issue.

- Gain-offset is UNCONFIRMED (Kimbell/Arrowsmith position). Default OFF
  (`gainOffsetPct: 0`). Requires CPA validation before enabling.

- IRMAA now fires 2yrs after ANY taxable disposition, not just 6th St
  (previous behavior was 6th-specific + suppressed if sixthMTR).

- Old "sellYear" is now derived from `dispositions.sixth.year` for shim
  compatibility with existing UI. The one-time-decisions "Sell 6th St"
  slider drives the sixth disposition; setting year > 2046 flips mode to
  'keep' automatically.
