--- 1. IO -> P&I MORTGAGE STRUCTURE (regression fix) ---

The regression: v3 amortized 6th/15th from 2021 via remainBal AND switched
IO->P&I payments on HI-debt clearing -- but the IO period is a LOAN TERM,
not a strategy toggle. Restored per-mortgage params in engine.js:

  MORTGAGE_DEFAULTS = {
    sixth:     { balance:805_495, rate:0.04875, originYear:2021,
                 originMonth:7, termYears:30, ioYears:10 },
    fifteenth: { balance:347_601, rate:0.0435, ... same shape },
  }

(balances calibrated from servicer statements, Jul 2026; IO payments derive
exactly: 805,495 x 4.875%/12 = $3,272/mo; 347,601 x 4.35%/12 = $1,260/mo).
Behavior, implemented as monthly STATE in both engines (annual buildScenario
inner loop + wfData):
  - during IO the balance stays flat absent extra principal;
  - at month 120 (~Jul 2031) the payment recasts to amortize the ACTUAL
    remaining balance over the remaining term: $5,260 (6th) / $2,171 (15th)
    at current balances -- extra principal before 2031 lowers the recast;
  - timeline + monthly Events emit "6th St mortgage: IO->P&I (+$1,988/mo)"
    and "15th St ... (+$911/mo)" FROM ENGINE STATE (rows carry
    mtgTransitions/mtgPayoffs); sold-before-2031 runs never recast;
  - closed-form helpers (mortgageBalanceClosed / mortgagePaymentClosed /
    mortgageMonthsSince) drive sale payoffs (computeDispo), the legacy
    6th St Sale Details panel, and NW balances now come from state
    (rows: primBalRaw/dplxBalRaw/lafBalRaw; chartData liq-NW + pin liq-NW
    use them instead of remainBal).
Escrow/tax/insurance stay in their existing cost lines (no double count).
Lafayette keeps the old amortizing treatment (BASE.lafPnI).
BASE.primIO/duplxIO/primPnI/duplxPnI are now legacy constants (unused).

KNOCK-ON (correct, not a bug): P&I forced from mid-2031 raises fixed costs
~$2.9K/mo; default HI-debt clear slips (annual ~2036). The old model
under-stated post-2031 mortgage cost by paying IO while HI debt existed.

EDGE (documented): disposition payoffs use the no-extra-principal closed
form -- bucket prepayments before a sale slightly overstate payoff /
understate proceeds (conservative).

--- 2. DEFAULTS TAB ---

New tab (Simulator | Cash Flow | Defaults | Map | Glossary): 7 groups /
~55 no-slider constants enumerated from code -- BASE property values/basis,
MORTGAGE_DEFAULTS (all fields), income baselines, health premiums +
transition years, per-property tax/ins, core living, DISPO_DEFAULTS +
liq-view cap-gains rates. Each row: label, numeric input, amber modified
dot + "code: X" hint. Reset per group + Reset ALL. Export/Import writes a
standalone defaults JSON (deliberately separate from pin files).

Persistence: localStorage key 'retirement_sim_defaults_overrides', applied
in the useState INITIALIZER (before the first engine run -- no flash).
Application: applyDefaultsOverrides(ov) in engine.js resets BASE /
MORTGAGE_DEFAULTS / DISPO_DEFAULTS to code values (captured in *_CODE
copies) then merges numeric overrides -- both engines read these objects
live; a defaultsRev counter is a dep of liveParams + wfData, and saved-pin
rows rebuild on change. Precedence (documented in the tab header): a
loaded pin's paramSnapshot wins for the sc params it contains;
defaults-tab overrides fill everything else. getDefaultsCode() exposes
pristine values for the modified-indicators.

--- 3. MORTGAGE-PRINCIPAL WATERFALL BUCKET ---

New sc params: mtgPrincipalOn (default OFF), mtgPrincipalCap ($2K/mo),
mtgPrincipalUncapped. Controls live in the Cash Flow waterfall panel
("Mortgage Principal Paydown", between Tier 3 and Tier 4) with an
applied-over-horizon + end-balances readout.

Position in the monthly chain: after maint/RD/OB buckets and the HI debt
sweep, immediately BEFORE the surplus->savings sweep. Fed by leftover
debt-sweep (avalanche fully satisfied first) plus what would otherwise go
to savings; capped per month; 6th (4.875%) before 15th (4.35%); only while
the property is held (Lafayette excluded by design). Conservation: the
bucket only diverts money that existed in the flow; OFF is bit-identical
to pre-v3.4.0 behavior (verified in node). Extra principal during IO
lowers the 2031 recast (verified: cap $3K under payOffHI -> recast delta
$1,988 -> $1,961). Early payoff emits "mortgage paid off early" event.
Annual mirror: leftover xtra or post-debt monthly surplus feeds the bucket
in the inner loop; the amount taken from surplus is subtracted from
annualSweepToSav; the v3.2.0 sale-remainder also passes through the bucket
before landing in sweep savings. The annual loop was restructured so
loans/mortgages/bucket step even under payOffHI.
