--- 1. SETTLEMENT CARD REORDER ---

New top-to-bottom flow ("show the number first, then decide how to use it"):
  1. Settlement need / year / gain-offset sliders + Kimbell note +
     same-year bump toggle (unchanged)
  2. Proceeds & Residual breakdown box (new, itemized -- see below)
  3. HI paydown % of residual slider
  4. Paydown-result line: "$X to HI debt (avalanche), $Y to invested"
The old three-line summary box (proceeds/need/residual + paydown line) is
replaced by items 2 and 4.

--- 2. ITEMIZED PROCEEDS BREAKDOWN ---

One collapsible section per disposition landing in the settlement year, all
values pulled directly from the disposeAsset return objects via a new
settleData useMemo (liveRows.dispoResults + dispoResultsNoOffset). NOTHING
is recomputed in the UI.

Per-property derivation (sell_taxable / home):
  Gross sale price -> − Selling costs (n%) -> = Net sale -> − Mortgage
  payoff -> = Pre-tax proceeds; then Recognized gain (offset-free figure)
  -> − Settlement gain offset (only when offset actually reduced this
  property's gain) -> = Taxable gain -> tax component line (recapture /
  fed / CA / CO, credit, same-yr bump share when present) -> − Total tax
  with "(would be $X without offset)" comparison from dispoResultsNoOffset
  -> = After-tax proceeds.
Partial 1031: mortgage payoff -> Cash boot (taxable slice) -> tax -> after-
tax = boot − tax. Full 1031: single line "no cash proceeds; $X gain
deferred".

Totals: Σ after-tax proceeds -> − Settlement need -> = Residual, plus a
shortfall note when proceeds don't cover the need (cash floors at 0).

Disclosure: each property header row toggles open/closed (▾/▸). Default:
expanded when exactly one property sells that year, collapsed otherwise
(settleOpen state).

The bump share shown in the tax line is derived as totalTax − (recapture +
fed + CA + CO) so the component line always sums to Total tax.

--- 3. DEV-MODE CONSISTENCY GUARD ---

useEffect (DEV only, via import.meta.env.DEV) console.warns when:
  - netSale != grossPrice − sellingCosts (per property)
  - after-tax proceeds != pre-tax − totalTax (boot − tax for partial 1031)
  - residual != max(0, Σ afterTaxNetProceeds − settlementNeed)
Guards against the UI drifting from the engine again (the v3.1.1 $919K bug
class). Silent in production builds.
