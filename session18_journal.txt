--- WHAT SHIPPED ---

v3.3.0 -- 6th St income segments-only: mode selector removed, overlaps
allowed and summed, auto-stop removed.

--- 1. MODE SELECTOR REMOVED ---

Deleted: the None/MTR/STR toggle, the flat "STR gross $/mo" +
start/stop-year sliders, and the params sixthIncomeMode / sixthSTRMonthly /
sixthSTRSchedule / sixthSTRStartYear / sixthSTRStopYear from
DEFAULTS/SC_DEFAULTS and the UI. sixthIncomeSegments is the SOLE driving
input; empty list = no 6th St income. Also removed as dead UI driven by the
old mode: the "6th St (while keeping / before sell)" Live-in/MTR toggles,
the "6th St MTR fallback rate/months" sliders, and the whole "6th St MTR
Schedule" editor (mtrSchedule no longer feeds the engine anywhere; the
field itself remains in saved scenarios, ignored). The card is renamed
"6th St Income (Segments)".

Back-compat -- migrateSixthIncomeSegments(snap) in defaults.js, applied in
makeParams (engine-level) and on pin load (restorePin / switchToPin;
buildRowsFromSnapshot gets it via makeParams):
  - explicit non-empty sixthIncomeSegments always wins;
  - mode 'mtr' (or the older sixthMTR:true shim) -> one flat MTR segment
    2026-2046 from the pin's rate x months (sixthMTRRent/sixthRent,
    sixthMTRMonths/sixthMonths);
  - mode 'str' -> one STR segment over [startYear, stopYear-1] with inner
    [{days:360, rate:monthly, type:'monthly'}] -- (360/30) x rate == 12 x
    monthly, so income is IDENTICAL to the old flat path (Q2: $8,370/mo net
    at 7% costs; MTR $5,000/mo -- exact).
  - An empty segments array does NOT suppress conversion (v3.1.1+ pins all
    carry an empty array next to their mode).
  - NOT converted: mtrSchedule / sixthSTRSchedule detail entries -- the
    conversion uses the flat values only, per spec. A legacy pin that relied
    on those schedule entries will get flat income instead (edge case;
    rebuild as segments by hand).

--- 2. OVERLAPPING SEGMENTS SUM ---

Engine (both annual buildScenario and monthly wfData): each year sums
income across ALL segments covering that year -- room STR + whole-house MTR
concurrently is valid. Cost treatment per kind unchanged (STR nets
platform+cleaning+mgr; MTR/LTR gross in annual, mgr netted in wfData).
validateSixthSegments no longer emits overlap errors -- only the
per-segment inner caps (Σ days <= 365, Σ months <= 12, clamped in
sixthSegmentGross as before). New engine helper sixthSegmentOverlaps(segs)
returns contiguous ranges covered by 2+ segments with the combined nominal
gross $/yr; the UI shows it as a neutral blue info line
(data-testid="sixth-seg-overlap-info"), never blocking.

--- 3. AUTO-STOP REMOVED ---

sixthSTRStopOnDebtClear param, the Auto-stop/Continue toggle, and the
debt-clear gating in both engines are gone. Segment year ranges are the
sole start/stop control. The field is ignored silently on old pin load
(Q2 asserts a run with the field === without). Test M8 (which asserted the
auto-stop behavior) deleted with the feature.
